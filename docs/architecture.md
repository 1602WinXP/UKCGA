# UKCGA 架构说明

> 本文档记录 UKCGA 的概念设计思路，所有内容均为假设性探讨，未经实验验证。

## 设计动机

当前大模型个性化方案的主要局限：

| 方案 | 局限 |
|------|------|
| 完整微调 | 参数规模大、破坏通用能力、版本锁定 |
| LoRA/Adapter | 仍需配合特定模型版本、多用户场景复杂 |
| 提示工程 | 表达能力受限、上下文长度限制 |
| RAG | 推理成本高、检索质量不稳定 |

UKCGA 试图探索一种不同的思路：**能否将用户知识表示为对基础模型的"修正信号"，而非独立的知识存储？**

## 核心假设

### 假设 1：差分表示的有效性
> 用户知识可以表示为"基础模型预测"与"期望输出"之间的语义差异。

**支持论据**：
- 基础模型已具备大量世界知识
- 用户知识往往是对通用知识的个性化调整
- 差异可能比完整知识更易压缩

**质疑点**：
- 复杂个性化是否都能表示为简单差异？
- 差异表示是否稳定跨版本？

### 假设 2：注意力掩码的表达能力
> 通过精细设计的注意力掩码，可以在不修改参数的情况下显著改变模型行为。

**支持论据**：
- 注意力机制是 Transformer 的核心
- 掩码可以精确控制信息流
- 已有研究表明注意力模式与特定能力相关

**质疑点**：
- 掩码能否实现细粒度的知识注入？
- 对模型能力的实际影响程度？

### 假设 3：元学习的跨用户迁移
> 存在通用的"用户适应策略"，可以跨用户迁移。

**支持论据**：
- 用户个性化存在一定共性（如偏好简洁/详细回答）
- 少量示例即可推断用户模式

**质疑点**：
- 用户差异是否过大导致元学习失效？
- 隐私与泛化的平衡？

## 待探索问题

### 技术层面
1. **差异编码的具体形式**
   - 向量？矩阵？更高维结构？
   - 如何量化"语义差异"？

2. **掩码的设计空间**
   - 稀疏度与表达能力的权衡
   - 多掩码的叠加与冲突

3. **版本兼容性机制**
   - 如何验证跨版本有效性？
   - 版本差异的补偿策略？

### 应用层面
1. **适用场景边界**
   - 哪些类型的用户知识适合此方案？
   - 哪些场景下传统方案更优？

2. **隐私与安全**
   - 差分表示是否泄露敏感信息？
   - 遗忘是否真的"彻底"？

## 可能的验证路径

### 阶段 1：概念验证
- 在小型模型上测试差分编码的可行性
- 验证简单场景下的版本迁移效果

### 阶段 2：组件验证
- 独立测试 SAM、DKE、MLA 各组件
- 分析各自的贡献和局限

### 阶段 3：系统集成
- 整合各组件形成完整流程
- 在真实场景下评估效果

### 阶段 4：理论分析
- 分析机制的理论边界
- 形式化证明或反证

## 开放问题

1. 这个方向是否值得深入探索？
2. 是否有被忽视的关键障碍？
3. 与现有方案的本质区别是什么？
4. 什么证据会证明或证伪这个思路？
